### Critical finding on batch mode

The hypothesis from research ("gpt-image-2 supports up to 8 coherent images from a single prompt with character continuity") is technically true but the model interprets a multi-panel prompt as "produce a single image containing N sub-panels" rather than "produce N separate images that together form a multi-panel storyboard."

Each of the 5 batch outputs is a 1024×1024 image with 4-5 mini-panels of Matt-like characters in unrelated scenes (mountains, gym, kitchen, sunset, library). The captions don't render because there is no canvas space dedicated to them — the model treats the request as a collage layout.

The 0.990 face score is artifact: the vision scorer compares overall visual similarity of two collages, not slide-to-slide character continuity. Two collages of mini-Matts read as "same person ≥ 0.90" even though the underlying carousel architecture failed.

**Batch mode is unusable for production carousels.** It either needs a fundamentally different prompt structure (one that explicitly asks for "N independent full-canvas images each rendering one caption") or the n>1 path needs to be abandoned for this use case.

### Why anchor wins

- Caption fidelity preserved (each call has a single caption to render, and the lead-with-text structure is unchanged)
- Per-slide scene fidelity preserved (each call has its own ROLE_BEAT and scene block)
- Face-crop pre-pass on the previous slide removes backdrop bleed risk that doomed earlier multi-angle ref tests
- Cumulative identity lock: by slide 5, the model has 5 anchors (source-ref + 4 prior-slide face crops in chain) all pulling toward the same identity

## Decision

Selected default mode: **anchor-chain**

Fallback shipped behind `--mode` flag: **per-slide** (current control behavior; lower face lock but lower latency, no ImageMagick dep)

Batch mode shipped behind `--mode batch` but marked **experimental — known to produce multi-panel grids rather than distinct slides on this prompt structure**. Documented for future research on alternative multi-panel prompt formulations.

Rationale: anchor mode delivers the only combination of (a) cross-slide face lock, (b) per-slide scene control, and (c) caption fidelity. The control mode preserves (b) and (c) but loses (a) — that's the original problem we set out to solve.

## Recommended next-step modifications to Phase 1

The plan currently treats batch as a viable default. Update Phase 1 to:

1. Default `--mode` is `anchor-chain` (was `<DEFAULT_FROM_PHASE_0>`)
2. `--mode batch` is shipped but documented as experimental with a warning printed to stderr when used
3. Recommend running the diagnostic per-character before adopting any non-anchor mode for production
4. Future work item: investigate prompt structures that successfully produce N independent full-canvas images via n>1 (e.g. "each image is a standalone 1024x1024 canvas, NOT a multi-panel grid")
